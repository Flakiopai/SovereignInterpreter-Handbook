# SovereignInterpreter Handbook v1.0.0 — Sovereign Edition

This repo documents existing interpreter behavior only. It does not ship runtime code.

## Doctrine alignment notes

Aligned with SovereignInterpreter v1.0.0 doctrine:

- Independent fork; not affiliated with upstream
- **Policy sandbox, not OS Seatbelt** — `safe` / `strict` / `full`
- **Kill-switch** mandatory by default (`.kill_switch` → `KillSwitchError`)
- **Universal fence rule** — model fences never auto-run without intent / confirm / `%run`
- **`allow_cloud: false`** — non-local LLM URLs rejected
- Memory packs are **local-only, cloud-free** (`LocalEmbeddings` + JSON on disk)

## REPL UX alignment notes

Aligned with live CLI / `display.py` behavior:

- Labels: `[system]` / `[model]` / `[confirm]` / `[run]` / `[console]` / `[skip]` / `[error]`
- Soft ANSI on tag only (`NO_COLOR` respected)
- Dim SI wordmark + boxed **Ready** banner (`model=` / `endpoint=` / `sandbox=` / `auto_run=`)
- Confirm: `[confirm] {lang}` + dim boxed full-code preview + `Run this code? [y/N]:`
- Thinking timer: `[model] thinking…` then `[model] thinking… (Ns)`
- Magics status lines use `[system]`; blocked `!` → `[error]` + `[system]` tip to `%sandbox full`
- Multi-line `"""` … `"""`; one-shot `sovereigninterpreter run "…"`

## Safety model alignment notes

Aligned with SovereignInterpreter v1.0.0 safety stack:

- Cloud URL gate + kill-switch + `SafetyRules` + confirm-first fence rule
- Sandbox denials → `[skip]` only (no misleading `[run]`)
- Typed envelopes remapped to REPL `[error]` / `[skip]` (including `KillSwitchError`, `CloudForbiddenError`, `SafetyViolation`, …)
- Intent helpers (`looks_like_user_code`, `user_requests_execution`) remain the local authorization surface
